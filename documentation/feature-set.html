Release Information

Project: USCIs

Functions

  • Data Set
    We have collected 178 different kinds of data for each school. This data is collected from several different databases to make sure our data visualization system can deliver the most meaningful information. Because our functions are based on the dataset, we will first describe our project by introducing the information that will be included in our system.
    Basic information: the data set contains most of the basic information about each school, including school classification (private or public), geographic data including cities, states, zip codes, and GPS coordinates, as well as other information that the user may be interested in.
    Financial data: the data set contains the major financial information for each institute including revenues, debt, funding, funds from local government, federal government, and more. It also contains information on tuition and fees, as well as awards and scholarships that the institute offers.
    Education data: this includes the types of certificates and degrees the school offers, the total number of degrees granted from the school and SAT/ACT scores for admission.
    Campus population: this describes the student body and has information on race, academic level, award and aid received, and many more. It also includes employee data, but we are not including it because this system is mainly to be used by students.
  • Core Functions
    Core functions are the most important functionalities of our project that ensure our system is able to help users clearly understand the information of each school. All core functions were expected to be completed when the project is finished.
  • General College Information
    The most basic function of the USCI system is to display general information about a school. Users will be able to select a school by clicking on it on the map or searching for it. After a user selects a school, there will be some basic information displayed including: university name, address, phone number, university type.
  • Fully Customizable Visualization
    For each school, we will implement the “fully customizable visualization” which is an upgrade of the basic information function. The user can select which information he or she wants to be displayed. Users can also configure how to visualize the data. For example, a user can choose to display the ratio of asian and white students and display them in a pie chart. Users can choose any of the values we have in the system and have them displayed in a window. The system will also suggest some visualization configurations to the user.
  • Comparing with other Schools
    The user will be able to select several schools to be compared. The system will allow a user to select the data and how to visualize the comparison. Most of the data can be selected in this function. For example, the user can select student population of 5 different schools and display this information using a bar chart.
  • Map Based Data Visualization
    The project's visualizations will function primarily on a map. We collected the GPS coordinates of each institute from a database “Post-Secondary University Survey 2010”7. Every university in our database has a GPS coordinate associated with it, which allows us to show them on a map with their exact locations.
    The different types of data will be visualized using colors. The system will display data on the map using different colors. For example, if the user wants to visualize degrees granted on the map, the area with more degrees granted will have a deeper color whereas the area with less degrees granted will have a lighter color. There will be no color shown in the area where there is no college.
  • Extra functionality
    There are several features we are considering implementing if we have extra time to work on the project. These functions include visualizing data by states, which will display the data on a map for each individual state in the United States. Instead of getting the numbers for each individual institution, it will get the totals for each state. For example, the data can be the total number of institutes in the state, total number of degrees granted in a year. These values would also be visualized using colors.
    We have also considered implementing data visualizations for the same university or area in different years. This might be possible since our data sets include data collected in different years. The main challenge is in cleaning the data because some universities are missing data for certain years.
    Because of the leaving of one member in our team. We have not implemented any of the extra functions.
  • User Interface
    We will create a web-based system. The system can be accessed through any web browser. It will function as an interactive web page with the functions described above.
    The user will be able to click on most data visualized on the web page and receive a response. There will be more details displayed for each data. The website will also display a navigation panel that allows the user to switch between different functions in the system.

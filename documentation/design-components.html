C-00: Data Handler

Description: The dataset first read one or more datasets into the Data Cleaning Module. If there are more than one datasets, the module first merges them into one dataset. Most of the datasets share one same value: the Unique Identification Number assigned by the IPEDS. So most of the time we can use this value to locate the same school and merge datasets into one. Then, the module will perform sorting based on the needs of the system. Most of time the data will be sort in the order of the Unique ID Number. The module will also sort the data based on other value if the module is generating a subset.
Environmental Constraints: Python
Available Interfaces:

Python command-lines

Excel

C-01: Visualization

Description: The visualization component is basically based on JavaScript. The browser on the local user computer will process this component. It has four modules: Base module, Information module, comparison module, and map module.Because D3 requires all functions dealing with the dataset to be implemented with in the dataset loading function, the data will be read directly by the three visualization modules but not the base module. The base module will print most of the figures that are not relative to the data, like the user interfaces. It also has functions that print figures for the visualization component but no connection to the data. These functions will be called by the visualization component, for example the function that print the x and y axis for a scatterplot. The Information module implements the "Information Visualization"; the comparison module will implements the comparison between schools; the map module will realize visualization of data on the map.
Environmental Constraints: JavaScript/D3
Available Interfaces:

Web browser

Netbean

Excel
